## Release Schedule

Lokus follows a regular release cycle:

- **Major releases** (x.0) - Every 6-8 months, introducing significant new features
- **Minor releases** (x.y) - Every 2-3 months, adding features and improvements
- **Patch releases** (x.y.z) - As needed, focusing on bug fixes and security

## Upgrade Guide

### From v1.2 to v1.3

v1.3 introduces several new features but maintains backward compatibility with v1.2 workspaces.

**What's Changed:**
- New YAML frontmatter support for Bases (optional)
- OAuth configuration for Gmail (re-authentication required)
- Performance improvements (no action needed)

**Action Required:**
1. Review OAuth settings if using Gmail integration
2. Optional: Add YAML frontmatter to notes for Bases features
3. Recommended: Review new MCP server capabilities

### From v1.1 to v1.2

Mostly backward compatible. Plugin developers should review the new plugin API.

**Action Required:**
1. Update plugins to use new API (if applicable)
2. Review template syntax changes

## Deprecation Policy

Lokus follows a clear deprecation policy to ensure smooth upgrades:

1. **Deprecation Notice** - Feature marked as deprecated, but still functional
2. **Grace Period** - At least two major versions before removal
3. **Migration Guide** - Provided when features are deprecated
4. **Removal** - Only after grace period and adequate notice

## Security Updates

Security updates are released as soon as possible after identification.

- **Critical vulnerabilities** - Patch within 24-48 hours
- **High severity** - Patch within 1 week
- **Medium/Low severity** - Included in next scheduled release

[Report security issues ‚Üí](https://github.com/lokus-ai/lokus/security)
