# CLI Reference

The `lokus-plugin` CLI is your primary tool for creating, building, and publishing plugins.

<Callout type="info">
  **Quick Install:** `npm install -g @lokus/plugin-cli`
</Callout>

## Quick Reference

| Command | Description |
| :--- | :--- |
| `create` | Scaffold a new plugin project |
| `dev` | Start development server with hot-reload |
| `link` | Symlink plugin for local development |
| `publish` | Publish to the Lokus Registry |

---

## Commands

<Tabs items={['Basics', 'Development', 'Publishing', 'Utility']}>

  <Tabs.Tab>
    ### `create`
    Scaffold a new plugin project.

    ```bash
    lokus-plugin create <name> [options]
    ```

    **Options:**
    - `-y, --yes`: Quick mode (skip prompts)
    - `--template <name>`: `basic`, `ui`, `editor`

    ### `install`
    Install a plugin locally.

    ```bash
    lokus-plugin install <path|id>
    ```
  </Tabs.Tab>

  <Tabs.Tab>
    ### `dev`
    Start the development server with hot-reloading.

    ```bash
    lokus-plugin dev
    ```

    <Callout type="warning" emoji="⚠️">
      **Tip:** Run `lokus-plugin link` first to symlink your plugin to the app!
    </Callout>

    ### `link`
    Symlink the current plugin to Lokus for local development.

    ```bash
    lokus-plugin link [--force]
    ```

    ### `test`
    Run the test suite (Vitest/Jest).

    ```bash
    lokus-plugin test [--watch] [--coverage]
    ```
  </Tabs.Tab>

  <Tabs.Tab>
    ### `login`
    Authenticate with the registry.

    ```bash
    lokus-plugin login
    # Follow the interactive prompts
    ```

    ### `publish`
    Publish your plugin to the registry.

    ```bash
    lokus-plugin publish
    ```

    **Options:**
    - `--tag <tag>`: `latest`, `beta`, `alpha`
    - `--dry-run`: Simulate without uploading

    ### `package`
    Build a `.vsix` file for manual distribution.

    ```bash
    lokus-plugin package
    ```
  </Tabs.Tab>

  <Tabs.Tab>
    ### `docs`
    Generate documentation website.
    
    ```bash
    lokus-plugin docs --serve
    ```

    ### `validate`
    Check your `plugin.json` and code for errors.

    ```bash
    lokus-plugin validate --strict
    ```

    ### `info`
    Show details about a plugin.

    ```bash
    lokus-plugin info <id>
    ```
  </Tabs.Tab>

</Tabs>

## CI/CD Integration

<Callout type="info">
  Use the `LOKUS_REGISTRY_TOKEN` environment variable in CI environments.
</Callout>

```yaml filename="github-actions.yml"
steps:
  - run: npm install
  - run: npm run build
  - run: npm run publish
    env:
      LOKUS_REGISTRY_TOKEN: ${{ secrets.LOKUS_TOKEN }}
```
