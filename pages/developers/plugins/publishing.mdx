# Publishing Plugins

Share your plugin with the world in 3 simple steps.

<Steps>

### Prepare for Release

Ensure your `plugin.json` has the correct version and metadata.

```bash
# Bump version
npm version patch

# Run validation
lokus-plugin validate --strict
```

### Authenticate

Generate a token from your **[Dashboard](https://registry.lokus.dev/dashboard)** and login.

```bash
lokus-plugin login
# Paste your token (starts with lpt_)
```

### Publish

Push your plugin to the registry.

```bash
lokus-plugin publish
```

<Callout type="success">
  **Done!** Your plugin is now live at `https://registry.lokus.dev/plugins/<your-plugin-id>`
</Callout>

</Steps>

## Advanced Publishing

### Release Channels

You can publish to different channels (tags) for testing:

```bash
# Beta channel
lokus-plugin publish --tag beta

# Alpha channel
lokus-plugin publish --tag alpha
```

Users can install these specific versions:
```bash
lokus-plugin install my-plugin@beta
```

### CI/CD Automation

Automate publishing with GitHub Actions:

```yaml
- name: Publish
  run: npm run publish
  env:
    LOKUS_REGISTRY_TOKEN: ${{ secrets.LOKUS_TOKEN }}
```

## Best Practices

*   **Semantic Versioning**: Use `MAJOR.MINOR.PATCH`.
*   **Changelog**: Keep `CHANGELOG.md` updated.
*   **Readme**: Include screenshots and clear usage instructions.
